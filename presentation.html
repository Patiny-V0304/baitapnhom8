THỰC HIỆN

Bước 1: Đăng Nhập & Lưu Trữ 2 Token

  • Cấu hình CredentialsProvider trong NextAuth
  • Khi đăng nhập thành công, nhận về 2 token từ backend (giả lập):

    • accessToken: Thời gian sống 60 giây

    • refreshToken: Thời gian sống 1 ngày

  • Bắt buộc can thiệp vào callback jwt() để:

    • Lưu accessToken, refreshToken, role vào JWT

    • Lưu accessTokenExpires (thời điểm token hết hạn)

Các thông tin đăng nhập để test:

• Username: student | Password: 123456 | Role: ROLE_STUDENT

• Username: advisor | Password: 123456 | Role: ROLE_ADVISOR

Bước 2: Phân Quyền Truy Cập

  • Trang Dashboard (/) chỉ cho phép người dùng có ROLE_ADVISOR truy cập
  • Nếu đăng nhập với ROLE_STUDENT → Hiển thị lỗi "Bị Từ Chối Truy Cập"
  • Nếu không đăng nhập → Redirect về trang /login

Bước 3: Demo Token Hết Hạn (Kịch Tính Nhất)

  • Trên Dashboard, có nút "Lấy danh sách lớp" (giả lập gọi API backend)
  • Nút này cần accessToken hợp lệ để hoạt động

Cách demo kịch tích (quan trọng):
  • 1. Đăng nhập với tài khoản advisor / 123456
  • 2. Bấm nút "Lấy danh sách lớp" → Thành công
  • 3. Chờ 60+ giây hoặc chờ "Access Token hết hạn sau: Xs" đến 0s
  • 4. Bấm lại nút "Lấy danh sách lớp" → Lúc này:
    • ⚙️ NextAuth tự động phát hiện Date.now() > accessTokenExpires
    • 🔄 NextAuth gọi hàm refreshAccessToken() sử dụng refreshToken
    • 🔑 Nhận accessToken mới từ backend (giả lập)
    • ✅ Cập nhật session với token mới
    • 📋 Cho phép gọi API thành công
  • 5. Mở Developer Console (F12) → Xem log: Token hết hạn, đang refresh...
Quan trọng:

Người dùng hoàn toàn không biết việc refresh token đang xảy ra dưới nền!
